# Руководство пользователя

**Интеллектуальная поисковая система по внутренней базе знаний университета**

Версия 1.0 · Учебная практика БПИ24

---

## 1. Назначение системы

Система позволяет:

- загружать документы в форматах **PDF** и **DOCX**;
- автоматически извлекать из них текст и индексировать его;
- выполнять **полнотекстовый поиск** по всем загруженным документам;
- видеть найденные фрагменты с **подсветкой совпадений** и оценкой
  релевантности.

---

## 2. Начало работы

### 2.1. Запуск системы

Если система ещё не запущена, выполните в корне проекта:

```bash
cp .env.example .env
docker compose up --build
```

После запуска откройте в браузере веб-интерфейс: **http://localhost:3000**

Интерфейс состоит из двух разделов, переключаемых в верхнем меню:

- **Загрузка** — добавление документов в базу знаний;
- **Поиск** — поиск по загруженным документам.

---

## 3. Загрузка документов

1. Перейдите в раздел **«Загрузка»**.
2. **Перетащите** один или несколько файлов в выделенную область
   *(«Перетащите файлы сюда…»)* — либо **щёлкните** по ней и выберите файлы
   через диалог. Поддерживается множественная загрузка.
3. Для каждого файла отобразится индикатор статуса:

   | Статус | Значение |
   |--------|----------|
   | **Загрузка...** | Файл передаётся на сервер |
   | **Индексация...** | Извлекается текст и строится поисковый индекс |
   | **Готово** | Документ доступен для поиска |
   | **Ошибка** | Файл не удалось обработать (см. сообщение) |

> **Ограничения:** допустимы только PDF и DOCX размером до **20 МБ**.
> Файлы другого формата или превышающие лимит будут отклонены с пояснением.

### 3.1. Список загруженных документов

Ниже зоны загрузки расположена таблица со всеми документами: название, дата
загрузки, размер и статус. Кнопка **«Обновить»** перечитывает список, кнопка
**«Удалить»** убирает документ из базы и поискового индекса.

---

## 4. Поиск

1. Перейдите в раздел **«Поиск»**.
2. Введите запрос в поле ввода.
3. Запустите поиск — нажмите кнопку **«Найти»** или клавишу **Enter**.

### 4.1. Результаты поиска

Результаты выводятся в виде карточек, отсортированных по релевантности.
В каждой карточке:

- **название файла**, в котором найден фрагмент;
- **номер страницы**;
- **фрагмент текста** с **жёлтой подсветкой** слов, совпавших с запросом;
- **оценка релевантности** (чем выше, тем точнее совпадение).

Над результатами показано общее число найденных фрагментов и время выполнения
запроса. Пометка «из кеша» означает, что ответ выдан мгновенно из кеша
повторного запроса.

### 4.2. Постраничный просмотр

Если результатов больше 10, внизу появляется навигация **«← Назад / Вперёд →»**
с указанием текущей страницы.

### 4.3. Если ничего не найдено

Появится сообщение: *«По вашему запросу ничего не найдено. Попробуйте изменить
формулировку»*. Рекомендации:

- проверьте опечатки;
- используйте более общие формулировки;
- убедитесь, что нужные документы загружены и имеют статус **«Готово»**.

---

## 5. Советы по эффективному поиску

- Поиск учитывает **морфологию русского языка**: запрос «обучение» найдёт и
  «обучения», «обучению» и т. п.
- Можно искать по нескольким словам — система найдёт наиболее релевантные
  фрагменты, содержащие эти слова.
- Поддерживаются запросы на русском и английском языках.

---

## 6. Возможные проблемы

| Симптом | Причина и решение |
|---------|-------------------|
| Файл не загружается, статус «Ошибка» | Неверный формат/размер или повреждённый файл. Используйте PDF/DOCX до 20 МБ. |
| Документ загружен, но не находится | Дождитесь статуса «Готово»; проверьте формулировку запроса. |
| «По запросу ничего не найдено» | Измените формулировку или загрузите релевантные документы. |
| Страница не открывается | Убедитесь, что система запущена (`docker compose up`). |
